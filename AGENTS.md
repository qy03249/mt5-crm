# 项目开发约定

## 数据库脚本

- 所有数据库迁移脚本、初始化脚本、种子数据脚本必须包含中文注释。
- 表、字段、索引、初始化数据涉及业务含义时，应写清楚中文说明。
- MySQL 表和字段应尽量使用 `comment` 描述业务含义。
- 资金、审核、MT5 写操作相关字段必须在注释中说明用途，避免后续误用。
- 新增数据库脚本前，先查看 `docs/data-field-map.md`，保证字段命名和页面字段一致。

## 配置原则

- 系统名称、Logo、favicon、语言、入出金规则、支付/收款方式、MT5 参数都应做成后台配置。
- 不允许把可变业务参数写死在代码中。
- `.env` 只用于本地环境，不提交真实密钥或密码。
